# E2E（Playwright）規約

- 要素の特定はユーザーに見える情報（ロール + アクセシブルネーム、表示テキスト、ラベル）で行い、`data-testid` などのテスト専用属性で特定しない
  - 理由: `data-testid` は本番 UI に現れずユーザー体験を反映しないため、その属性が壊れても実害のある UI 崩れを検知できない。ロールやアクセシブルネームで特定すれば、E2E が実際のユーザー操作・アクセシビリティの担保も兼ねられる
  - 指針: `getByRole` / `getByText` / `getByLabel` を優先する。一意に絞り込めない場合は、要素に付ける `aria-label` 等をユーザーにとって意味のある名前にして特定する（例: 記事カードの開くトリガに `記事「<タイトル>」を開く`）
  - 注意: 特定のためだけに `data-*` 属性を足さない。名前が衝突する場合は、テスト都合ではなくアクセシビリティ上も妥当な命名で解消する
