# 記入例

テンプレート別の記入例。いずれも「差分から読める詳細は省き、課題と方針に絞る」「箇条書きは粒度をそろえ、トップ＝サマリ／子＝根拠」を満たしている。選んだテンプレートの例を1つ読んでから書くとよい。

## バグ以外の改善（リファクタ・基盤整備）

```markdown
# 概要
## 課題
契約違反の顕在化が手書きの `if (...) throw new Error(...)` に散在し、書き味・検出・型の絞り込みがばらついていた（#1002）。

## 修正内容
- fix: #1002
- 契約アサートを `@trend-diary/common/contract` の `assert` / `assertNonNull` に集約した
  - 手書きの条件チェック＋throw をなくし、`asserts` 型注釈で以降の narrowing も効くようにした
- 契約違反は専用の `AssertionError` で識別し、errorHandler が 5xx として通知する
- 実装は `node:assert` に依存せず自前とした
  - `common` はクライアントからもインポートされ、`node:assert` はブラウザバンドルを壊すため
```

トップの各バレットは「集約した」「識別する」「自前とした」と同じ方針レベルの粒度でそろえ、根拠は子バレットへ落としている。移行ファイルの列挙やテスト追加の詳細は書かない。

## バグフィックス

```markdown
# 概要
## 問題のあった領域

- [x] バックエンド
  - [x] データアクセス（DB・SQL）

### 要因の詳細
未ログインで `read_status=0` を指定すると記事一覧が 500 になっていた。既読判定サブクエリは `active_user_id` を bind パラメータ前提にしているが、未ログイン時はこの値が未束縛のままクエリへ組み込まれ、プレースホルダ数に対して bind 値が不足し D1 が例外を投げていた。

### 再現手順
1. ログアウト状態にする
2. `GET /api/articles?read_status=0` を実行する
3. 200 ではなく 500 が返る

## 修正方針
- fix: #955
- 未ログイン時は既読判定サブクエリ自体を組み立てずスキップする
  - bind 値とプレースホルダ数が常に整合する

### その方針を選んだ理由
既読・未読はログインユーザーに紐づく状態で、未ログイン時は判定対象が存在しない。ダミーの `active_user_id` を束縛する案は、実在しない値で判定の意味が曖昧になるため採らず、サブクエリを生成しない方針とした。

## その他、参考情報
特になし
```

要因・再現手順は diff から読めない証跡なので残す。テンプレの「問題のあった領域」チェックボックスは埋めてよい（受け入れ基準の丸写しとは別物）。
