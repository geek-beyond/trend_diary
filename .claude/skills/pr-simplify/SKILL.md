# PR Simplify

PRを作成（または push）したタイミングで、その PR の差分に対して `/simplify` を自動実行する。
レビュワーを待たずに、過剰設計・YAGNI 違反・不要な防御コードを最小変更で削ぐのが目的。

レビューコメントの抽出・返信・スレッド resolve は **このスキルの責務ではない**。
それらはレビュー対応フローの仕事であり、ここでは一切扱わない。

## いつ使うか

PR 作成というイベントに紐づけて simplify を回すのがこのスキルの狙い。
**simplify を明示されていなくても**、以下のときは必ず発火させる:

- PR を作成・ドラフト作成した（「PR 作って」「PR 出した」だけでも対象）
- PR ブランチに push した直後
- 「PR 出したから simplify して」と言われたとき

逆に、PR 作成と無関係な文脈では発火しない（下記「やらないこと」参照）。
PR を作るたびに毎回手動で `/simplify` を叩かなくても済むようにするのが目的。

## 手順

### 1. 対象を特定する

- 直近作成した PR、または明示された PR 番号 / ブランチを対象にする。
- `git diff <base>...HEAD` で simplify をかける差分の範囲を把握する。

### 2. `/simplify` を実行する

- `/simplify` を実行する。
- `/code-review` の指摘のうち **simplify 対象**（過剰設計 / YAGNI 違反 / 現在の
  呼び出し側で発火しない防御コード）を選び抜き、最小コード変更で解消する。
- 取捨選択の判断基準とコミット規約は `simplify` スキルに従う（足すのではなく引く）。

### 3. 変更が入ったら検証する

- simplify で差分が生じた場合のみ、リポジトリ規約に従って format / lint / test を実行する。
- 失敗したら原因を切り分け、再実行して結果を確定させる。

### 4. コミット・push

- 変更ファイルだけ `git add` する（`git add -A` や `.` は使わない）。
- 何を simplify したかが分かる簡潔なコミットメッセージで記録する。
- PR ブランチへ push する。

### 5. 結果を報告

- どの指摘を **採用 / 不採用 / 部分採用** したかを可視化する（`simplify` スキルの
  「出力の Discipline」に従う）。
- simplify する対象が無ければ「削るものは無かった」と明示して終える。空コミットは作らない。

## やらないこと

- レビューコメントの抽出・コミット ID 返信・スレッド resolve（このスキルの責務外）。
- 新機能の追加、将来のための防御コードや TODO の埋め込み（`simplify` の反パターン）。

## 自動実行（pre-push フック）

このスキルを毎回手で叩かなくても済むよう、push 時に simplify を確定実行する
git フックを用意している（GitHub Actions 不要・ローカル完結）。

- 実体: リポジトリ直下の `.githooks/pre-push`
- 有効化: `mise run setup-hooks`（= `git config core.hooksPath .githooks`）
- 挙動: push 対象の差分へ `claude -p "/simplify ..."` をかけ、simplify が変更を
  加えた場合は push を中断して確認・コミットを促す（簡素化前のコードを push させない）。
- 無限ループ防止に `PR_SIMPLIFY_RUNNING` で再入をガードしている。

`claude` CLI が無い環境（CI 等）ではフックは何もせずスキップする。
