# ユーザー向け文言の言語規約

- コード中のユーザー向け文言（UIに表示されるラベル・ボタン文言・エラーメッセージ・トースト・バリデーションメッセージ・通知文・メール本文など、エンドユーザーの目に触れるテキスト）は常に日本語で記述する
  - 対象は特定のファイル種別に限定しない。`.ts` / `.tsx` のコンポーネントや文言定義、i18n のメッセージカタログ、サーバー側のエラーメッセージ、通知・メールのテンプレートなど、ユーザーに見えるあらゆる箇所に適用する
  - 理由: 本プロジェクトのエンドユーザーは日本語話者を前提としており、UI上の言語を統一して一貫した利用体験を提供するため
- 適用対象外（英語のままで構わないもの）:
  - ソースコードの識別子（変数名・関数名・型名・ファイル名など）
  - ログ出力・例外の内部メッセージなど、開発者のみが目にするテキスト
  - 外部API・ライブラリが要求するキーや列挙値など、仕様上英語が必要なもの
- 多言語対応（i18n）の仕組みがある場合は、ユーザー向け文言を直書きせず日本語の翻訳リソースとして登録する
