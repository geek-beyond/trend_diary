# テスト規約

- 分岐や関数が増えない範囲で、なるべくテーブル駆動テスト（`it.each` 等）にする
  - 理由: 同種のケースを表形式で一覧でき、ケース追加が容易で網羅性を読み取りやすい
  - 注意: テーブル化のために条件分岐やケースごとのコールバック関数が増えるなら、可読性を優先して個別のテストに分ける
- 外部APIへの実通信は禁止し、必ずモックする
  - 理由: 実通信はネットワークや外部サービスの状態に依存して flaky になり、レート制限・課金・副作用も招く。テストは決定的で高速に再現できる必要がある
  - 方法: Vitest の `vi.stubGlobal('fetch', vi.fn())` や `vi.mock()` で外部呼び出しを差し替え、想定するレスポンス・エラーをテスト側で制御する
