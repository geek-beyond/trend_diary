# cron worker の独立パッケージ化

Status: Accepted

Relevant PR: #681

# Context

cron worker は `rss-parser` / `drizzle-orm` / `common` / `datastore` / `domain` / `notification` しか使わないにもかかわらず、React・Radix・recharts・storybook などフロントエンド依存をすべて含む単一の `package.json`（`my-app`）からビルド・デプロイされていた。

そのため以下の課題があった。

- cron のデプロイ・テスト時に不要なフロントエンド依存までインストールされる
- 「cron は common/datastore/domain/notification だけに依存する」という境界がコード上では保証されておらず、誤って React 系を import しても気づけない

すでに `common` / `datastore` / `domain` / `notification` は `packages/*` 配下のワークスペースパッケージへ切り出し済みであり、同じ方針で cron も独立パッケージ化できる状態だった。

# Decision

cron worker を `application/packages/cron`（`@trend-diary/cron`）として独立したワークスペースパッケージに切り出す。

- エントリポイント: `packages/cron/src/worker.ts`
- 依存は `@trend-diary/common` / `@trend-diary/datastore` / `@trend-diary/domain` / `@trend-diary/notification` と `drizzle-orm`(catalog) / `rss-parser` のみを宣言する
- wrangler 設定はパッケージ配下の `packages/cron/wrangler.toml` に移設し、デプロイは `wrangler deploy --config packages/cron/wrangler.toml` で行う
- テストは miniflare (`@cloudflare/vitest-pool-workers`) を使う特殊環境のため、パッケージ専用の `vitest.config.ts` を持ち、`pnpm --filter @trend-diary/cron test` で実行する
- migrations はアプリケーション全体で集中管理しているため、パッケージからは `../../migrations` を参照する

## Reason

- パッケージの `dependencies` を最小限に宣言することで「cron はバックエンド依存だけに依存する」という境界を package.json レベルで明示できる
- フロントエンド依存（React 系）が cron の依存グラフから物理的に外れる
- cron 専用 CI（`cron-test` / `cron-deploy`）が他パッケージと同じ `pnpm --filter` の枠組みに揃う

# Consequences

## 分離前後の比較（記録）

`pnpm exec wrangler deploy --dry-run` および production 依存クロージャの計測値。

### バンドルサイズ（wrangler dry-run）

| | 分離前（`my-app` からビルド） | 分離後（`@trend-diary/cron`） |
| --- | --- | --- |
| Total Upload | 602.74 KiB | 603.99 KiB |
| gzip | 106.48 KiB | 106.50 KiB |

バンドルサイズはほぼ不変。esbuild が単一エントリからのビルド時点で未使用のフロントエンド依存をすでに tree-shaking していたため。バンドル面での縮小効果は小さい。

### 依存数（production 依存の推移的クロージャ）

| | 直接依存 | 推移的クロージャ（外部パッケージ） |
| --- | --- | --- |
| 分離前（`my-app`） | 38 | 149 |
| 分離後（`@trend-diary/cron`） | 6 | 31 |

直接依存は 38 → 6、推移的な外部依存は 149 → 31 へと約 8 割削減。インストール・デプロイ時に解決される依存グラフが大幅にスリム化した。

### React 系フロント依存の除外

分離後の cron 依存クロージャに `react` / `react-dom` / `react-router` / `@radix-ui/*` / `recharts` / `storybook` 等のフロントエンド依存は **含まれない**（0 件）。

なお `domain` が認証用に `@supabase/supabase-js` を宣言しているため、cron の依存クロージャには Supabase SDK が推移的に含まれる。これは Issue #681 が許可する依存（`→ common, datastore, domain, notification`）の範囲内であり、実行時バンドルでは tree-shaking により除外される。

## その他

- `my-app` の `dependencies` から `rss-parser` を除去（cron 専用依存のため）
- ルートの `wrangler.cron.toml` と `src/test/vitest/cron/config.ts` を廃止し、設定をパッケージへ集約
- `cron-test` / `cron-deploy` ワークフローのパスフィルタ・実行コマンドをパッケージ構成に追従
